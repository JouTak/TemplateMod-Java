# JouTak Mod Template (Java Edition)

Заготовка для мода на JouTak (ДжоуТек) ([joutak.ru](https://joutak.ru))


# Перед использованием

1. Отредактируйте:
    - `gradle.properties` &mdash; укажите **версию** и **название** мода, а также **ссылку на репозиторий** с вашим модом
    - `src/main/java/ru/joutak/` &mdash; переименуйте **основной класс** под название вашего мода
    - `src/main/resources/fabric.mod.json` &mdash; укажите **правильный путь** до основного класса мода и укажите себя как **автора**
    - `gradle/libs.versions.toml` &mdash; измените **версии** **JDK**, **Minecraft**, **Fabric** и **других библиотек** по необходимости
    - Укажите **полный путь до папки `mods/`** тестового клиента Minecraft в переменной окружения системы `MODS_PATH`

> [!TIP]
> Указав путь до папки `mods/` тестового клиента, можно собирать мод с помощью конфигурации `[Snapshot] Build`, тогда jar-архив с вашим модом сразу будет создаваться в нужной папке автоматически.


# Сборка мода

1. Открыть проект в **IntelliJ IDEA**
2. Нажать кнопку `Run` справа сверху для сборки:
    - `[Dev] Run Client` &mdash; запускает Minecraft-клиент прямо из IDE с загруженным модом (для разработки)
    - `[Snapshot]` &mdash; создает архив с модом в папке `mods/` тестового клиента (путь до которого нужно указать в переменной окружения `MODS_PATH`)
    - `[Release]` &mdash; создает архив с модом в папке `build/` проекта


# Перед началом выполнения тестового задания

> [!NOTE]
> Данный параграф предназначен для тех, кто захотел попробовать себя в роли разработчика модов для Minecraft в составе команды JouTak.

Базовое описание, что нужно сделать:
1) Сделать собственный **fork** данного репозитория
2) Склонировать созданный репозиторий себе по **SSH** (`git clone`)
3) Открыть проект в любой удобной для вас **IDE**

> [!TIP]
> Рекомендуем использовать **IntelliJ IDEA**, так как в ней есть полноценная поддержка **Java** со всеми фишками и плюшками, но выбор за вами

4) Разобраться с расположением папочек и синтаксисом **Java**, затем прописать базовую логику, которая была описана в вашем задании в issues (Любая креативность поощряется)
5) Собрать мод с помощью **Gradle**

> [!TIP]
> Конфиги уже заранее прописаны (один для разработки, один для тестовых целей и один для полноценного релиза), вам самим ничего прописывать не надо. Для разработки лучше использовать `[Dev] Run Client`, а для тестов на реальном клиенте &mdash; `[Snapshot]` версию, чтобы каждый раз не перекидывать вручную мод в папку клиента, главное не забудьте создать переменную окружения `MODS_PATH`!

6) Запустить свой мод через конфигурацию `[Dev] Run Client` или подключить к своему локальному Minecraft-клиенту
7) Проверить, что мод работает исправно
8) Отправить **pull request** на **GitHub** и дождаться ответа

> [!TIP]
> Если долго никто не отвечает, то можно пингануть кого-нибудь, чтобы реквест отсмотрели и приняли :)

Если мод написан без базовых ошибок (синтаксических и т. п.), то на **GitHub**'е автоматически соберется такой же jar-архив, который будет так же работать на твоем локальном Minecraft-клиенте.

Поначалу всё можно писать в основном классе, но потом все вещи по типу команд, обработчиков событий и различной логики нужно выделить в отдельные классы и пакеты.

> [!IMPORTANT]
> Главное, если есть какие-то вопросы &mdash; не стесняйся спрашивать!
